Why Are There So Many Machine Learning Classifiers?

Posted on Fri 22 December 2017 in Data Science • Tagged with Data Science

I'm sure that at some point, most data scientists must ask the question: Why are there so many classifiers? There's k-Nearest Neighbors, Logistic Regression, SVMs, Decision Trees, Naive Bayes, Random Forests, Neural Networks, and the list goes on! But do we really need all of these classifiers?

Well, I'd say it probably isn't really necessary to have all of these classifiers, but they are nice to have. There are some algorithms which arguably usually don't perform as well others for many datasets. These usually include KNN and Decision Trees. These are usually considered "weak learners", but they are useful because they are easily interpretable and they can help you make sense of the data and maybe see some relationships that are not as obvious when using other classifiers.

Principal Component Analysis

Posted on Fri 13 January 2017 in Data Science • Tagged with Data Science

Principal Component Analysis is sort of a tricky topic, but it's incredibly useful to strengthen your statistical analysis.

Before diving into PCA, I first want to dicuss a bit about what it is and why it's important. PCA is a form of dimensionality reduction. When working on a data science problem, you usually have a set of features to analyze and you might use those features to make predictions. More often than not, however, many of your features are superfluous. They do not add a lot of predictive value, and keeping them in your model could lead to overfitting and describing noise rather than signal. With dimensionality reduction, you can reduce the number of features you use to build your model the the goal of improving the fit.
